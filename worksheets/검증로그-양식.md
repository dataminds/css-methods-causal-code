# 검증 로그 양식 (통계판)

> 이 책의 내기: 분석 코드는 AI가 짜 주되, 그 코드를 믿을지는 연구자가 검증으로 판정한다.
> 검증은 전부 코드로 한다. 손계산은 검증 수단이 아니다(개념을 처음 배울 때 감을 잡는 용도까지만).
> 상세 = 책 부록 B.

## 검증 3종

| 검사 | 물음 | 방법 |
|---|---|---|
| 1. 독립 재계산 | 같은 답이 다른 길로도 나오나 | 같은 양을 다른 코드 경로로 계산해 대조 |
| 2. 극단값 시험 | 답을 아는 자료에서 그 답이 나오나 | 효과를 0으로 만든 자료(뒤섞기·상수화·복사본)를 먹여 예측값 확인 |
| 3. 재실행 | 같은 씨앗에서 같은 수가 나오나 | 씨앗 고정 후 처음부터 재실행해 전 수치 대조 |

## 로그 항목 (분석 하나당 다섯 칸)

```
[분석] 무엇을 구했나
[AI]   무엇을 시켰고 무엇을 받았나 (프롬프트 요지 + 받은 코드 요지 한 줄)
[검사1] 독립 재계산: 무엇 ↔ 무엇, 결과
[검사2] 극단값: 어떤 가짜 자료, 예측, 결과
[검사3] 재실행: 씨앗, 결과
[판정]  채택 / 수정 후 채택(무엇을 고쳤나) / 기각(왜)
```

## 기입 예 (S1 주 분석)

```
[분석] 실험판 주 분석: 정제 표본(366)에서 개입/통제의 사후 mil 비교
[AI]   "journey_exp.csv에서 attn_1 통과자만으로 cond별 mil 평균·SD·t검정" 지시 ;
       pandas 필터 + scipy ttest_ind 코드 수령
[검사1] t검정 p=.028 ↔ 뒤섞기 5,000회 p=.030 — 근사 일치, 채택 방향
[검사2] cond를 rng(73)으로 뒤섞은 영세계 → 예측: 차이≈0, p 큼 → 실제 p=.346. 통과
[검사3] 씨앗 73 전체 재실행 → 5.26/4.98/t=2.21 전부 재현. 통과
[판정]  채택
```

## 자주 새는 곳

- 표본이 다르다(정제 전 자료로 계산 ; N부터 대조).
- 자유도·꼬리 방향이 다르다(단측/양측·등분산). p가 정확히 절반이거나 두 배면 이것부터 의심.
- 재실행마다 수치가 바뀐다 = 씨앗 없는 무작위가 어딘가에 있다.
- 반올림 단계가 다르다(중간 반올림 vs 마지막). 대조는 반올림 전 값으로.
